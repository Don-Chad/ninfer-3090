# M2.8 P1 Q5090 Canonical Conv1d Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox
> (`- [ ]`) syntax for tracking.

**Goal:** Make the M2.8/M3 canonical q5090 GDN conv1d layout real in converter output, fixtures, runtime
binding, tests, docs, and regenerated real-weight artifacts.

**Architecture:** Move the one-time GDN conv1d layout conversion from model bind time to the offline q5090
converter. The canonical q5090 tensor remains `linear_attn.conv1d.weight` with logical/stored shape
`[conv_dim, gdn_conv_k, 1] = [10240,4,1]`, but its payload bytes must be runtime-native: tap-major with
channel as the fastest varying element, so the CUDA conv1d kernels can read `weight[k * C + c]` directly.
Runtime binding then accepts only the canonical shape and returns a direct non-owning view.

**Tech Stack:** Python 3 converter tooling, PyTorch/safetensors, C++17, CUDA, CMake/CTest, q5090
`qwen3_6_27b` model card.

## Critical Layout Rule

Do not implement P1 as a plain `reshape=(10240,4,1)`.

HF raw conv1d is `[C,1,K]` with PyTorch/safetensors row-major bytes:

```text
raw byte order: c0k0, c0k1, c0k2, c0k3, c1k0, c1k1, ...
```

The runtime conv1d kernels read:

```cpp
weight[c]
weight[C + c]
weight[2 * C + c]
weight[3 * C + c]
```

So canonical q5090 must store:

```text
runtime-native byte order: k0c0, k0c1, ..., k0c10239, k1c0, ...
```

The metadata shape must be `[10240,4,1]`, and the direct runtime view must be `[10240,4]`.

Add the transform function:

```python
def runtime_native_gdn_conv1d(t: "torch.Tensor"):
    if t.dim() != 3 or t.shape[1] != 1:
        raise ValueError(f"gdn conv1d expected [C,1,K], got {tuple(t.shape)}")
    c = int(t.shape[0])
    k = int(t.shape[2])
    if k != 4:
        raise ValueError(f"gdn conv1d expected kernel width 4, got {k}")

    # q5090 Tensor uses dim0-fastest runtime order. The returned tensor deliberately
    # has metadata shape [C,K,1] while its flat bytes are [K,C].
    return t[:, 0, :].transpose(0, 1).contiguous().reshape(c, k, 1)
```
